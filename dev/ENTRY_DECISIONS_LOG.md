# Entry Decisions Log

## Overview 
This file tracks decisions about adding, refining, or rejecting data layer entries.

It serves two purposes:

prevent redundant or low-quality additions
maintain consistency in how concepts are evaluated
---

## How To Use This File
When proposing a new idea:

Add it under Under Review

When a decision is made:

Move it to:

Accepted
Refined
Rejected

Do NOT duplicate entries across sections.

Keep entries short and scannable.

### Quick Rules

- If unsure → leave in Under Review
- If redundant → move to Refined
- If combined → move to Rejected
- If clearly distinct → move to Accepted
---


## [Under Review]
Use this section to propose new concepts before evaluation.

Format:

concept_name
Type: (core / system / environment)
Notes: short description or intent
Submitted: YYYY-MM-DD


Example
corruption_track
Type: system
Notes: long-term cost/benefit progression tied to player choices
Submitted: 2026-04-22
